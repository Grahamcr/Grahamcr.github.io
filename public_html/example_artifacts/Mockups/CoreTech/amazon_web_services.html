API Gateway


Amazon API Gateway is a fully managed service that makes it easy for developers to create, publish, maintain, monitor, and secure APIs at any scale. With a few clicks in the AWS Management Console, you can create an API that acts as a “front door” for applications to access data, business logic, or functionality from your back-end services, such as workloads running on Amazon Elastic Compute Cloud (Amazon EC2), code running on AWS Lambda, or any Web application. Amazon API Gateway handles all the tasks involved in accepting and processing up to hundreds of thousands of concurrent API calls, including traffic management, authorization and access control, monitoring, and API version management.


Amazon API Gateway service is enabled by default for all AWS user roles unless otherwise restricted by your Business Unit. Please contact your Business Unit Enterprise Architect for access questions.


Autoscaling


With Autoscaling, you can scale your EC2 automatically for no additional charge. This service is well-suited for applications that experience variability in usage, as it scales up/down your servers as your traffic changes. This provides elastic capacity, cost savings, and multiple availability zones.


CloudFront


CloudFront is an easy and cost effective way to distribute content to end users who have low latency, high data transfer needs. CloudFront delivers your entire website using the AWS global network of edge locations.


CloudWatch Dashboards


Amazon CloudWatch Dashboards enable you to create re-usable graphs of AWS resources and custom metrics so you can quickly monitor operational status and identify issues at a glance. With CloudWatch Dashboards, you can monitor operational metrics with custom graphs of your AWS resources from a single pane of glass. You can create text-based or graphical widgets, add custom text annotations and links to graphs, change the time range, resize and reorganize widgets, and reuse and share dashboards.


Direct Connect


With AWS Direct Connect, your applications have a dedicated network connection from GE premises to AWS. This service reduces network costs, increases bandwidth throughput, and provides a more consistent network experience.


DynamoDB


DynamoDB is a no-sequence database that allows you to store any mount of data with no limits with fast, predictable performance. DynamoDB is easy to provision and change request capacity needed for each table. This is best used for a No SQL Database.


Elastic Beanstalk


Elastic Beanstalk service auto-handles your application deployment, and gives you full control over your AWS resources to power your applications. Because of its easy-to-use set-up and ability to scale web applications, it is perfect for users who have no need or desire to stand up their infrastructure.


 Elastic Block Storage (EBS)


EBS is a file system that provides you with persistent block level storage that is replicated within your Availability Zones. Likewise, EBS takes snapshots that are stored in S3 that creates a hard drive for you to store your information. Snapshots do not disappear, and allow for a quick and easy recovery of your data if you accidentally delete it.


Elastic Compute Cloud (EC2)


Amazon’s Elastic Compute Cloud works as a resizable compute capacity. You can use Amazon EC2 to launch as many or as few virtual servers as you need, configure security and networking, and manage storage. Amazon EC2 enables you to scale up or down to handle changes in requirements or spikes in popularity, reducing your need to forecast traffic. This services gives you complete control of computing resources and reduces time required to obtain and boot new server instances to minutes.


EC2 lets you use web service interfaces to launch instances with a variety of operating systems, load them with your custom application environment, manage your network’s access permissions, and run your image using as many or few systems as you desire–including Windows and Linux. Likewise, you are only paying for what you use while you use it.


EC2 Instance Tagging allows you to tag common keys to help manage your environments. Key Pairs within EC2 are private, and act like a password. These keys identify a person or service that needs access to a secured resource.


Elasticache


Elasticache is a web service that makes it easier to set up and manage in-memory cache environments within the cloud. This provides high performance, resizable, and cost-effective in-memory cache, while removing complexity associated with deploying and managing cache environment and managing patching, cache node failure detection and recovery. ElastiCache does not require coding—rather, it is a simple lookup of queries to improve latency.


Elastic Load Balancers


AWS Elastic Load Balancers distribute incoming traffic across applications running on one or more EC2 instances, supports health checks to detect unhealthy EC2 instances, and supports routing and load balancing. This service looks for instances that have the least amount of work and distributes the works evenly.


Flowlogs


Flowlogs enable users to capture information about IP traffic going to and from network interfaces in your VPC.

Example use case for Flowlogs:


To troubleshoot why specific traffic is not reaching an instance

To monitor traffic reaching your instance

Glacier


Glacier provides an extremely secure, unlimited space for data that you don’t need but want to hold on to to be stored. This services offloads the administrative burdens of operating and scaling archival storage to AWS.


Identity Access Management (IAM)


AWS IAM is a two-part, self-service offering that gives the Business Unit teams the power to self-serve many of the IAM functions in AWS. It changes the mixed-policy setup from the core team to a “deny all prohibited actions” policy setup, with automated testing and auditing. The first level of this service allows Senior Enterprise Architects to manage their AWS API access to replace the SupportCentral AWS API Access Request Form. The second level of this service allows Senior Enterprise Architects to manage IAM Users, Group and Roles in the VPC.


Inspector


Inspector is an automated security assessment service that helps improve the security and compliance of applications deployed on AWS. It automatically assesses applications for vulnerabilities or deviations from best practices, and produces detailed repots with prioritized steps for remediation.


Key Management Service (KMS)



KMS allows GE cloud users to leverage encryption services to enhance security within their Virtual Private Clouds (VPC) when using other AWS services.


KMS uses encrypted applications and data that live in the VPC and enables you to use encryption keys from 1) your own application via AWS EDK, and 2) supported AWS services.


Kinesis Firehose


Kinesis Firehose is the easiest way to load streaming data into AWS. This service can capture and automatically load streaming into Amazon s3 and Amazon Redshift, enabling real-time analytics scales to match the throughput of your data. Firehose can batch, compress, and encrypt the data before loading, minimizing storage used and increasing security

S3


S3 service provides unlimited data storage in a highly scalable, reliable, fast, and durable environment. S3 places your data into buckets to hold your objects (or files and metadata that describes a file), and gives each object a unique URL for access control.


Snowball

Web Application Firewall (WAF)


AWS WAF lets you create rules to filter web traffic based on conditions that include IP addresses, HTTP headers, or custom URIs. This gives you an additional layer of protection from web attacks that attempt to exploit vulnerabilities in custom or third party web applications. In addition, AWS WAF makes it easy to create rules that block common web exploits like SQL injectionandcross site scripting. AWS WAF allows you to create a centralized set of rules that you can deploy across multiple websites. This means that in an environment with many websites and web applications you can create a single set of rules that you can reuse across applications rather than recreating that rule on every application you want to protect.


AWS Web Application Firewall service is enabled by default for all AWS user roles, unless otherwise restricted by your Business Unit. Please contact your Business Unit Enterprise Architect for access questions.
